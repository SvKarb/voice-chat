# 🎙 VoiceRoom — Голосовой чат на WebRTC

Курсовой проект: голосовой чат в реальном времени с использованием WebRTC, Node.js и Socket.io.

## Стек технологий

| Слой | Технология | Назначение |
|------|-----------|-----------|
| Frontend | HTML5 / CSS3 / Vanilla JS | Интерфейс пользователя |
| Сигнализация | Node.js + Socket.io | Обмен SDP и ICE |
| P2P связь | WebRTC (RTCPeerConnection) | Аудио-потоки между участниками |
| Сервер | Express.js | HTTP + статика |

## Архитектура

```
Участник A                  Сервер (Node.js)              Участник B
    │                            │                              │
    │── join-room ──────────────►│                              │
    │                            │◄──────────────── join-room ──│
    │◄── room-users ─────────────│                              │
    │                            │──── user-joined ────────────►│
    │                            │                              │
    │── offer ──────────────────►│──── offer ──────────────────►│
    │◄── answer ─────────────────│◄─── answer ──────────────────│
    │── ice-candidate ──────────►│──── ice-candidate ──────────►│
    │                            │                              │
    │◄══════════════ P2P аудио-поток (WebRTC) ════════════════►│
```

**Принцип работы:**
1. Сервер выступает только как **сигнальный** — передаёт SDP-офферы/ответы и ICE-кандидаты
2. После установки соединения аудио идёт напрямую **P2P** между браузерами (без сервера)
3. При входе нового участника уже присутствующие создают для него RTCPeerConnection

## Установка и запуск

```bash
# 1. Установить зависимости
npm install

# 2. Запустить сервер
npm start

# 3. Открыть в браузере
http://localhost:3000
```

Для разработки с авто-перезагрузкой:
```bash
npm run dev
```

## Функционал

- ✅ Создание / вход в именованную комнату
- ✅ Голосовая P2P связь в реальном времени (WebRTC)
- ✅ Поддержка нескольких участников (Mesh-топология)
- ✅ Mute / Unmute микрофона
- ✅ Визуальный список участников
- ✅ Уведомления о входе/выходе
- ✅ Генерация случайного ID комнаты

## Используемые WebRTC API

- `getUserMedia()` — захват аудио с микрофона
- `RTCPeerConnection` — P2P соединение
- `createOffer / createAnswer` — SDP-согласование
- `addIceCandidate` — NAT traversal
- `ontrack` — получение удалённого аудио-потока
